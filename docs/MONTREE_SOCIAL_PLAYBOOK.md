## HOW TO DRIVE THIS (the command + the protocol)

Say to WebClaude, e.g.:

> **"Upload video 1 — guru to all the relevant social media sites."**

WebClaude then does **Step A** (open tabs). Because the platforms block automated file-attach and we never want a surprise auto-post, you then drop the video file into each tab and say:

> **"Take the next step."**

WebClaude then does **Step B** (paste caption + hashtags + title where each platform needs them) and **stops at the final review screen on every tab. It never clicks Post / Share / Publish — you do that.**

### Step A — open the upload tabs (one per platform)
1. Instagram — https://www.instagram.com/ → "+" → **Reel**
2. Facebook — https://www.facebook.com/ → **Reels → Create reel**
3. TikTok — https://www.tiktok.com/upload
4. X — https://x.com/compose/post
5. LinkedIn — https://www.linkedin.com/feed/ → **Start a post → video**
6. YouTube — https://studio.youtube.com/ → **Create → Upload videos** (mark as **Short**; vertical 9:16 under 3 min auto-qualifies)

### Step B — for each tab
- Confirm the video is attached (you dropped it in).
- Paste that platform's caption / post body.
- Add the platform's hashtags.
- YouTube: paste **Title**, **Description**, and **Tags** separately.
- Put `[SIGNUP]` in the caption/body (IG + TikTok also rely on link-in-bio).
- **Stop at the pre-publish screen. Do not publish.** Report "Tab N ready for review."
- If a platform asks for login or a captcha, pause and let me handle it, then continue.

> Honest limits (so nothing silently fails): WebClaude **can** open tabs and fill text. It usually **cannot** attach the local video file for you (you drop it) and must **never** hit Post. Treat it as "everything filled, you review and publish."
