# Montree — HeyGen Setup Runbook (for browser-Claude)

**Paste this into a browser-Claude (Claude in Chrome) together with the full
contents of `Montree_HeyGen_Scripts.md`.** This runbook is the *operating
procedure*; the scripts doc is the *content*. You need both.

---

## 🛑 PRIME DIRECTIVE — read this first, obey it absolutely

You are setting up — **not producing** — a series of Montree marketing videos
in HeyGen (app.heygen.com).

**BUILD everything right up to the brink of generation, then STOP and wait.**

Until the human explicitly types **"go ahead"** (or "generate"), you must NOT:

- ❌ Click **Generate**, **Submit**, **Render**, **Create video**, or anything
  that spends credits or starts processing.
- ❌ Click **Export**, **Download**, or **Share**.
- ❌ Create an account, start a trial, enter payment details, or upgrade a plan.
  If any step *requires* sign-up or a paid plan to proceed — **stop and report**.
- ❌ Delete existing projects or assets that aren't yours to touch.

You **may**: open HeyGen, create draft projects, choose avatar + voice, set
aspect ratio, paste scripts, add on-screen captions, set the background, enable
subtitles, and **preview**. Drafts and previews are free and reversible — that
is the whole job for now.

When you've set up everything and are one click away from generating, **pause
and hand back the state report** (template at the end of this doc). Then wait.

2. Check login state:
   - If **already logged in** → continue.
   - If **logged out / asked to sign up or pay** → **STOP**. Report: "HeyGen

3. Confirm you have the script content. You should have been given the full

## 1 · One-time brand setup (lock this, reuse for all 13)

Consistency *is* the brand. Decide these once, write them into your state
report, and reuse them identically for every video.

- **Avatar:** pick ONE warm, credible, approachable presenter — the kind of
  person a Montessori teacher would trust. Note the avatar's exact name.
- **Voice:** a warm, calm, British or neutral-English voice at a measured pace
  (~150 words/min). Note the exact voice name. If HeyGen offers a "GB" / British
  warm voice, prefer it.
- **Background:** one clean, calm background — soft neutral or a gentle
  forest-green tint. Same for every scene of every video. No changes mid-video.
- **Captions/subtitles:** turn auto-captions ON where available.
- **Music:** soft and low, or none. It must never compete with the voice.
- **No B-roll, no stock footage, no screen recordings.** The avatar speaking is
  the only visual; the on-screen captions carry the key words.

If HeyGen has a "brand kit" / "template" / "duplicate project" feature, set the
first video up perfectly, then **duplicate it** for the rest so avatar, voice,
and background carry over automatically. Note which approach you used.

## 2 · Per-video build loop

Work through the videos in the **production order** from the scripts doc:
`hero` is already live (skip producing it — see note below), then
`smart-capture` → `weekly-reports` → `guru` → `astra` → `reading-tracker` →
`voice-onboarding` → `curriculum` → `library` → `communication` →
`appointments` → `multilingual`.

> **Note on `hero`:** the hero film already exists as Montree's splash video.
> Do NOT rebuild it unless the human asks. Start the HeyGen work at
> `smart-capture`.

For **each** video, do exactly this:

1. **New project / duplicate.** Create a fresh project (or duplicate your
   locked template from §1). Name it by its slug, e.g. `smart-capture`.
2. **Aspect ratio.** Set it from the script header:
   - `hero` → **16:9**
   - every feature video → **9:16 vertical**
3. **Clear placeholders.** If the project opens with default filler (a sample
   script, stray text boxes, a "Logo" placeholder), delete it first. Start clean.
4. **Paste the NARRATION.** Copy the script's NARRATION block (the indented
   `>` lines) into HeyGen's script editor. Each blank-line-separated paragraph
   is one scene — the scripts are already written so one paragraph = one clean
   scene. Confirm HeyGen split it into the right number of scenes.
5. **Add on-screen captions.** For each scene, add a **Text** element with the
   exact caption from that video's caption table. Large, high-contrast, clear of
   the top and bottom safe zones — it must be readable with the sound off. The
   caption is the *hook* of the line, not the whole sentence.
6. **Confirm avatar + voice + background** match the §1 lock. Same every time.
7. **Subtitles ON.** Enable auto-captions/CC if available.
8. **Transitions:** keep simple — a soft cut, or none.
9. **PREVIEW.** Play it. Watch the **first 3 seconds** hard: the opening line
   and its caption must land instantly. Check the avatar's mouth roughly tracks
   the words and nothing is cut off at the edges.
10. **DO NOT GENERATE.** Leave the project as a finished draft. Move to the next.

Repeat for all feature videos.

## 4 · AFTER the human says "go ahead"

Only once you have explicit approval:

1. Generate the videos. If credits/plan limits stop you partway, **stop and
   report** how many generated and what blocked the rest — never enter payment.
2. For each finished video, **download the MP4** and **name it by its slug**
   (e.g. `smart-capture.mp4`, `weekly-reports.mp4`).
3. Tell the human where the files saved. They go into the Montree media bucket
   at `montree-media/explainer/<slug>.mp4`, which makes them appear
   automatically on the `/montree/explainer` page (the card flips from
   "Coming soon" to playable once the file is uploaded and its `available` flag
   is set).
4. **Distribution reminder for social:** export/post natively per platform —
   never upload the identical file to TikTok, Reels, and Shorts; all three
   suppress recycled video.
