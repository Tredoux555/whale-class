## 1. VOICE PROFILE

The register is **"nursery trap": dark-trap 808 menace played completely straight against
preschool-innocent content, with a deadpan potato as the recurring anti-climax.** It is spooky-
playful, ritualistic, percussive, and built so the class is always a participant, never an
audience. Rhythm is call-and-response over a heavy 68 bpm beat; structure is a single repeating
frame that swaps one word at a time; every unit lands on the potato.

Evidence base: all 12 SATPIN song lyric sheets, the 27 Dark Phonics song titles (lessons 05тАУ31),
and the "Where Is Segina?" + "Sam Sat" + "The Cat Naps" + "Sit, Cat, Sit!" + "I Am an Ant" books.

### The 10 voice rules a writer MUST follow

1. **Rule of three тАФ repeat the payload word three times to close a beat.**
   The last line of every hook triples the target word. *"She's at the park, park, park!"* ┬╖
   *"It's a mat, mat, mat!"* ┬╖ *"POTATO! POTATO!"* The triple IS the rhythm; drop it and the song
   stops swinging.

2. **The Potato gag is mandatory and always last.** Teacher Potato (a deadpan potato in tiny
   round glasses) crashes the end of EVERY song and EVERY book as the anti-climax. In songs it
   drops loud after a whispered lull: *"Sam sat on a POTATO! (POTATO!)"*; *"It's a POTATO!"*
   In books the final spread leaves the noun **blank** and the class shouts "POTATO" from the
   picture (Segina spread 8тЖТ9; Sam spread 9 "On the mat?!"; Cat Naps spread 9 "The cat naps on aтАж?!").
   The one sanctioned variation: in the **p** week the potato finally starts with the target sound,
   so the gag flips тАФ the class shouts *"YES?!"* instead of "NO!" (*"A potato?! (YES?!) тАж but it's
   still not the answer!"*). The gag can bend, never be cut.

3. **One frame = one machine; swap exactly one word per verse/spread.** Each book and word-song is
   a single sentence frame run repeatedly with one slot changing: *"Where is Segina? She's at the
   ___."* / *"The cat naps on the ___."* / *"Sam sat on the ___."* / *"I am ___."* The teacherFocus
   notes say it explicitly: *"the song is a machine, the class feeds it"* and *"once they crack line
   one they read the rest themselves."* Never vary two things at once.

4. **Whisper тЖТ SHOUT dynamics; build a hush before the drop.** Every piece has a "[Breakdown тАФ
   whispered, bass drops out]" and a "(shhhтАж)" beat right before the loud payoff. *"She's at home.
   Shhh. Segina is asleepтАж WAKE UP, SEGINA!"* The suspense page is often text-only on a dark page
   (Segina spread 8: *"WhereтАж isтАж SeginaтАж?"*, no image). Contrast is the engine.

5. **Call-and-response тАФ the class is the second voice.** Write in an echo the children shout back,
   marked in parentheses: *"Where is Segina? (where? where?)"* ┬╖ *"It's a cat?! (a cat?!)"* ┬╖
   *"(oh no!)"*. Roughly every hook line has a bracketed class answer. The reader is never passive.

6. **Mild grossness / body humor is on-brand and welcome.** The Dark Phonics song titles set the
   ceiling: *"Pop Pop Puppy Poop"* (p), *"Big Baby Burp"* (b), *"Icky Sticky Pig"* (i),
   *"Tick-Tock Stinky Sock"* (t), *"Muddy Monkey"* (m), *"Yummy Bug in my Cup"* (u). Gross-cute,
   never mean, never scary-for-real. Stink, mud, poop, burps = yes; cruelty = no.

7. **Deadpan absurdity тАФ escalate a mundane premise past logic, played straight.** The cat naps on a
   map, then a cap, then inside a *frying pan*; the potato *teaches at a whiteboard*; the cat uses
   the potato *as a pillow*. The art note word is literally "deadpan." Comedy comes from committing
   to the absurd image with a straight face, not from jokes-about-jokes.

8. **Reversal / comeuppance тАФ the little one wins, the smug one gets got.** Sam sits on the cat тЖТ
   *"The cat sat on Sam!"* (cat perches on his head "like a hat, fair is fair"). The tiny ant is
   nearly squashed then *"pops out unharmed and triumphantтАж the smallest is the toughest"* (*"I can!
   I am an ant!"*). Bake a turn-of-the-tables into the middle of the arc.

9. **Leave the last noun blank тАФ the child fills it from the picture.** The climactic reveal is
   never printed; the page says *"Am I aтАж?!"* / *"It is aтАж?!"* / *"On the mat?!"* and the picture
   supplies the answer the class yells. This is a designed participation ritual, tied to the song's
   final shout. Every book must end on a blank-plus-picture.

10. **Sound-true, never schwa, never letter-name тАФ articulation honesty is a hard constraint.**
    The /s/ is a long snake hiss, *"NEVER 'ess' тАж and NEVER 'suh'."* /t/ is *"t-t-t not tuh."*
    /p/ is a *"quiet lip-pop тАж NEVER 'pee' and NEVER 'puh.'"* Suno lyrics respell vowels phonetically
    ("ih-ih-in" for i, "Ah-ah-ah" for a) so the machine never sings the letter name. Any new lyric
    or chant must honor this or it teaches the wrong thing.

**Two supporting rituals** (not standalone rules but load-bearing): **name-swap** (in circle,
Segina/"an ant" is replaced by each real child's name and real places тАФ *"I am Amy! I am Leo!"*),
and the **two-song shape** every week (a "sound" stutter-isolation song + a "word" sentence-frame
song; wk1 "a" ships only one because a is both sound and word).

### Ritual structure of a book (locked template, 9 spreads typical)
1. Cast/premise establishing spread (single subject, spotlit entrance).
2тАУ6. Frame repeated, one word swapped per spread; the running gag escalates.
   A reversal beat sits mid-arc (spread 5тАУ6).
7тАУ8. Resolution / truce / "at last" + calm group "victory" spread with the cast gathered.
9. **Suspense/blank spread тЖТ the Potato reveal** (noun omitted, class shouts it).
Read-aloud books (a, t) run ~9тАУ10 spreads and are allowed non-decodable vocab; decodable books
(s, p, i, n) keep every printed word inside the cumulative letter pool.

## 5. SCHEMA FACTS (WeekSpec, from types.ts) тАФ so redesigned readers drop straight in

### A book = `BookSpec`
```
BookSpec {
  title: string
  readAloud: boolean   // true = teacher-read/echo-read (weeks 1тАУ3 only); false = child-decodable
  spreads: BookSpread[]
  backCoverWords: string[]   // word list printed on back cover
}
BookSpread {
  n: number        // 1-based spread number
  text: string     // left-page (dark page) text; "" = wordless
  image: string    // lowercase WORD of the full-bleed right-page image; MUST match an assets[] entry
  artNote?: string // art-direction note beyond the base MJ prompt
}
```
- **`readAloud` semantics (the decodability escape hatch):** when `readAloud: true`, **book text is
  exempt from the decodability validator** (materials are still always checked). Comment says
  "weeks 1тАУ3 only." In the existing files, the a and t books are `readAloud: true` (they use
  non-decodable vocab like "chair/apple/supermarket"); the s, p, i, n books are `readAloud: false`
  and every printed word is inside the cumulative pool.
- **The `image` field must be a single lowercase word that resolves to an `assets[]` entry** (and
  to an `imageUrls` key). Multi-word images exist ("bus stop") but the asset filename must match.

### Decodability validator rules (as documented in types.ts + the plan)
- **Decodability (core rule):** book text + material words may use only **letters taught тЙд this
  week** + **glue words already introduced**. `readAloud:true` exempts BOOK TEXT ONLY.
- **One song = one pattern, ~10тАУ15 unique words.**
- **Every image referenced anywhere MUST appear in `assets[]`.**
- Validator specifics from the plan (`scripts/curriculum/validate-specs.mjs`): a `SPINE` array
  (line 64) hard-codes the letter order and decides decodability by a file's `week` number indexed
  into it; strict-checked fields are `book.spreads[].text` (non-readAloud weeks), `materials.sentences`,
  `materials.tracing.words`, `materials.dictionary`. Cast names (capitalized) are decodability-exempt;
  there are glue/picture-vocab allow-lists. Level-1 pattern registry is empty (no magic-e/suffix
  gates fire тЙд wk26). **Picture-vocab fields (threePartCards, matching, bingoPool, coloring) are a
  softer gate than strict text** тАФ they rarely block but should be tidied for pedagogy.

### Materials = `MaterialsSpec`
```
MaterialsSpec {
  threePartCards: string[]   // words on 3-part cards; EACH needs an image asset
  sentences: string[]        // full decodable sentences (strict-checked)
```
